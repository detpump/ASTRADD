# Ask Mode Guidance
- Documentation is scattered: architectural intent lives in [`docs/CODEBASE_ANALYSIS.md`](workspace/skills/aster-trading/docs/CODEBASE_ANALYSIS.md:1) while test plans sit in [`docs/TESTING_STRATEGY.md`](workspace/skills/aster-trading/docs/TESTING_STRATEGY.md:1). Always cite those files instead of README when explaining flows—they contain the real constraints (e.g., TP hierarchy, self-heal limits).
- Risk config numbers in `config/risk_config.json` aren’t authoritative; runtime clamps in [`trading_system.py`](workspace/skills/aster-trading/src/trading_system.py:1205) and [`risk_guard_v2.py`](workspace/skills/aster-trading/src/risk/risk_guard_v2.py:30) override them. When documenting limits mention both sources or clarifications will be wrong.
- Incident/rollback procedures exist only in [`docs/ROLLBACK_PROCEDURES.md`](workspace/skills/aster-trading/docs/ROLLBACK_PROCEDURES.md:15); there’s no separate runbook. Point operators there if they ask for recovery guidance.
